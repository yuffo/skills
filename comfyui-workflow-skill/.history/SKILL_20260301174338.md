## 必要信息
- 环境变量"COMFYUI_PATH"为comfyui资源目录
- 目录结构: $COMFYUI_PATH 下包含 custom_nodes/  input/  models/  output/  temp/  user/
- comfyui内嵌python scripts目录: $COMFYUI_PATH\.venv\Scripts
- 工作流位置: $COMFYUI_PATH/user/default/workflows/
**重要**: 脚本会自动扫描以下位置的内置节点：
- `$COMFYUI_PATH/nodes.py` (标准位置)
- `$COMFYUI_PATH/comfy/nodes.py` (comfy 子目录)

如果 `nodes.py` 不存在（仅有数据目录），内置节点将无法扫描，只能扫描 `custom_nodes/` 下的自定义节点。

## 工作流程
1. 根据需求规划节点和连接
2. 使用脚本获取所需节点信息 脚本提供查询所有节点名称和单个节点详情
3. 编写工作流 核对节点名称和输入输出

- 遇到未安装节点时,查询github到项目目录,使用git clone 自动安装节点到custom_nodes python依赖由用户启动comfyui自动安装 安装完成后 检查目录是否存在确保安装
- 简单的节点可以自行编写,自定义节点写到$COMFYUI_PATH/custom_nodes/comfyui-yuff目录 节点名使用中文
- 遇到需要指定软件版本难以兼容的项目 放弃 选择其他项目 要避免为了一个节点安装一个庞大的库 首选节点利用率高的库
- 通常github仓库中项目根目录下包含名称含workflow(不分大小写)的文件夹包含示例工作流 可以参考

## 工作流 JSON 格式规范

### 连线格式

**links 数组格式**:
```json
"links": [
  [link_id, source_node_id, source_slot_index, target_node_id, target_slot_index]
]
```

**示例**:
```json
"links": [
  [1, 1, 0, 5, 0],   // link_id=1: 节点1的输出slot0 → 节点5的输入slot0
  [2, 1, 1, 2, 0],   // link_id=2: 节点1的输出slot1 → 节点2的输入slot0
  [3, 1, 1, 3, 0]    // link_id=3: 节点1的输出slot1 → 节点3的输入slot0
]
```

**节点内部连接标记**:
- 输出节点使用 `"links": [link_id, ...]` 数组表示连接到哪些 link
- 输入节点使用 `"link": link_id` 表示连接到哪个 link

**完整节点示例**:
```json
{
  "id": 1,
  "type": "CheckpointLoaderSimple",
  "outputs": [
    {"name": "MODEL", "type": "MODEL", "links": [1], "slot_index": 0},
    {"name": "CLIP", "type": "CLIP", "links": [2, 3], "slot_index": 1},
    {"name": "VAE", "type": "VAE", "links": [7], "slot_index": 2}
  ]
}
```

### slot_index 说明
- `slot_index` 从 0 开始计数
- 对应节点输出/输入端口的顺序
- 例如 CheckpointLoaderSimple 输出: MODEL(0), CLIP(1), VAE(2)


## 脚本说明

### comfyui_nodes.py - 节点查询工具

通过 AST 静态解析 Python 文件获取节点信息，无需启动 ComfyUI，支持解析未安装依赖的新节点。

#### 命令

```bash
# 列出所有节点
python scripts/comfyui_nodes.py list

# 详细列出所有节点
python scripts/comfyui_nodes.py list -d

# 查看节点详情（包含输入输出参数）
python scripts/comfyui_nodes.py info <节点名称>

# 搜索节点
python scripts/comfyui_nodes.py search <关键词>

# 按输入类型筛选节点
python scripts/comfyui_nodes.py by-input IMAGE

# 按输出类型筛选节点
python scripts/comfyui_nodes.py by-output MASK

# 列出所有已知类型
python scripts/comfyui_nodes.py types

# JSON 格式输出（便于程序解析）
python scripts/comfyui_nodes.py list --json

# 强制重新扫描（禁用缓存）
python scripts/comfyui_nodes.py list --no-cache
```

#### 参数

| 参数 | 说明 | 默认值 |
|------|------|--------|
| `--comfyui-path, -p` | ComfyUI 安装路径 | F:/SD/comfy 或 COMFYUI_PATH |
| `--detailed, -d` | 显示详细信息 | - |
| `--json, -j` | JSON 格式输出 | - |
| `--no-cache` | 禁用缓存重新扫描 | - |

### comfyui_custom_nodes.py - 插件仓库查询

查询自定义节点的 GitHub 仓库地址，快速获取安装命令。

#### 命令

```bash
# 列出已安装插件
python scripts/comfyui_custom_nodes.py list

# 搜索节点仓库
python scripts/comfyui_custom_nodes.py search <关键词>

# 获取安装命令
python scripts/comfyui_custom_nodes.py install <插件名>
```

### github_helper.py - GitHub 项目浏览（减少 token 消耗）

轻量级 GitHub 项目浏览工具，只获取必要信息，减少 AI token 消耗。

#### 命令

```bash
# 列出项目根目录文件/文件夹
python scripts/github_helper.py ls <owner/repo>

# 读取 README.md 文件
python scripts/github_helper.py readme <owner/repo>

# 示例
python scripts/github_helper.py ls comfyanonymous/ComfyUI
python scripts/github_helper.py readme cubiq/ComfyUI_IPAdapter_plus

# JSON 格式输出
python scripts/github_helper.py ls cubiq/ComfyUI_IPAdapter_plus --json

# 指定分支
python scripts/github_helper.py readme owner/repo --branch dev

# 限制 README 行数（默认 200 行）
python scripts/github_helper.py readme owner/repo --max-lines 100
```

#### 参数

| 参数 | 说明 | 默认值 |
|------|------|--------|
| `--branch, -b` | 分支名称 | main |
| `--json, -j` | JSON 格式输出 | - |
| `--token, -t` | GitHub Token (提高 API 限制) | - |
| `--max-lines` | README 最大行数 | 200 |
